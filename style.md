# UI Style Guide

本项目计划建设一个“网页对话式视频分析界面”：用户先登录，登录后进入类似 GPT 的聊天工作台，用自然语言提交视频分析需求、查看分析过程和结构化结果。界面需要同时适配桌面端和手机端网页，整体保持现代、简洁、sleek、专业，不做复杂装饰，不做营销式首页。

本文档用于指导后续 agent 进行 UI 设计和前端实现。除非用户另有明确要求，所有页面、组件、文案和交互都应遵守本文风格。

## 1. 产品气质

### 关键词

- 现代、克制、清爽、轻量。
- 专业但不冰冷，适合高频使用。
- 类似 GPT 的对话工作流，但要更贴近“视频分析工具”的业务场景。
- 视觉上保持 sleek：干净的层级、细腻的阴影、明确的交互状态、足够留白。

### 避免

- 不要做花哨的营销落地页。
- 不要使用大面积渐变、装饰球、玻璃拟态泛滥、霓虹色、复杂插画背景。
- 不要让页面看起来像数据后台那样厚重拥挤。
- 不要使用过多卡片嵌套卡片。
- 不要在界面里放大量“功能说明文字”或教学文案。

## 2. 页面结构

### 登录页

登录页只保留必要功能，第一屏即登录体验，不做营销 hero。

推荐结构：

- 页面居中放置登录面板。
- 顶部可保留简洁品牌名或产品名。
- 面板内包含账号输入、密码输入、登录按钮、必要的错误提示。
- 如需要注册、忘记密码、验证码、第三方登录，应以轻量链接或次级按钮形式出现。

登录页视觉要求：

- 背景使用非常浅的冷灰或近白色，不使用复杂背景图。
- 登录面板宽度桌面端约 `360px - 420px`。
- 手机端面板不应像悬浮卡片一样拥挤，可变为全宽内容区，左右边距 `20px`。
- 输入框高度建议 `44px - 48px`。
- 主按钮高度建议 `44px - 48px`。

### 聊天页

聊天页是产品核心。它应像一个轻量、专注的视频分析工作台。

推荐结构：

- 桌面端：
  - 左侧窄侧栏：历史会话、新建对话、账号入口。
  - 中间主区域：消息流。
  - 底部固定输入区：输入文本、上传/粘贴视频链接、发送按钮。
- 手机端：
  - 默认隐藏侧栏，通过顶部菜单按钮打开抽屉。
  - 顶部只保留当前会话标题、菜单按钮、必要操作。
  - 输入区固定在底部，并兼容移动端安全区。

聊天页视觉要求：

- 主消息区域宽度要适中，桌面端正文容器建议最大宽度 `760px - 880px`。
- 不要把所有内容塞进厚重卡片。消息本身可以有轻微背景区分。
- 用户消息靠右或使用更明显的强调背景。
- AI/系统分析消息靠左或占据内容宽度，强调可读性。
- 视频分析结果如果包含结构化字段，应使用轻量分组、表格、标签或折叠块展示。

## 3. 色彩规范

整体色彩以中性灰、近白、深墨色为基础，使用一个冷静的品牌强调色。颜色应有足够对比度，适合长时间阅读。

### 推荐色板

基础色：

- 页面背景：`#F7F8FA`
- 面板背景：`#FFFFFF`
- 轻区块背景：`#F1F4F8`
- 分割线：`#E3E7EE`
- 主文本：`#111827`
- 次级文本：`#4B5563`
- 弱文本：`#8A94A6`

品牌强调色：

- 主色：`#2563EB`
- 主色 hover：`#1D4ED8`
- 主色 active：`#1E40AF`
- 浅强调背景：`#EAF1FF`

状态色：

- 成功：`#16A34A`
- 警告：`#D97706`
- 错误：`#DC2626`
- 信息：`#0284C7`

### 暗色模式可选

如果实现暗色模式，应保持同样克制，不要变成纯黑高对比霓虹风。

暗色推荐：

- 页面背景：`#0F141B`
- 面板背景：`#151B24`
- 轻区块背景：`#1C2430`
- 分割线：`#293241`
- 主文本：`#F3F6FA`
- 次级文本：`#B7C0CE`
- 弱文本：`#7F8A9A`
- 主色：`#60A5FA`

## 4. 字体规范

优先使用系统字体，避免额外加载沉重字体文件。

推荐 font-family：

```css
font-family:
  Inter,
  ui-sans-serif,
  system-ui,
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  "PingFang SC",
  "Microsoft YaHei",
  sans-serif;
```

字号建议：

- 页面主标题：`24px - 32px`，仅登录页或空状态可使用。
- 页面/会话标题：`16px - 18px`。
- 正文消息：`15px - 16px`。
- 辅助说明：`13px - 14px`。
- 标签、时间、状态：`12px - 13px`。

行高建议：

- 正文：`1.6 - 1.75`
- 标题：`1.25 - 1.35`
- 按钮和表单：`1.2 - 1.4`

字重建议：

- 正文：`400`
- 重要正文：`500`
- 标题：`600`
- 不要大面积使用 `700` 或更粗字重。

## 5. 间距与布局

使用统一 spacing scale，避免随意写间距。

推荐 spacing token：

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
--space-8: 32px;
--space-10: 40px;
--space-12: 48px;
```

布局规则：

- 桌面端页面左右安全边距至少 `24px`。
- 手机端页面左右安全边距建议 `16px - 20px`。
- 主要内容区不要贴边。
- 消息之间垂直间距建议 `16px - 24px`。
- 聊天输入区与消息列表之间保持清晰分隔。
- 固定底部输入框需要给消息列表预留 padding-bottom，避免遮挡最后一条消息。

## 6. 圆角、阴影与边框

整体圆角要现代但不过度圆润。

推荐：

- 小组件、按钮、输入框：`10px - 12px`
- 消息气泡：`14px - 18px`
- 面板、弹窗：`16px - 20px`
- 卡片最多 `8px` 的限制不适用于聊天气泡；但普通工具卡片应尽量控制在 `8px - 12px`

边框：

- 默认边框使用 `1px solid #E3E7EE`。
- 输入框 focus 使用品牌色描边或柔和 ring。

阴影：

```css
--shadow-sm: 0 1px 2px rgba(15, 23, 42, 0.06);
--shadow-md: 0 10px 30px rgba(15, 23, 42, 0.08);
--shadow-popover: 0 18px 48px rgba(15, 23, 42, 0.14);
```

使用原则：

- 大部分区域靠背景、边框、间距建立层级。
- 阴影只用于登录面板、弹窗、下拉菜单、底部输入区等需要浮起的元素。
- 不要让每个卡片都带明显阴影。

## 7. 组件规范

### 按钮

按钮类型：

- Primary：主要操作，如登录、发送、开始分析。
- Secondary：次级操作，如取消、导出、复制。
- Ghost/Icon：工具操作，如菜单、设置、上传、清空。

按钮要求：

- 主按钮背景使用品牌主色。
- 禁用状态要明显降低透明度，并禁止 hover 强调。
- 按钮文本要短，优先使用动词。
- 图标按钮必须有 hover/focus 状态和无障碍标签。
- 常见工具按钮优先使用图标，例如发送、上传、菜单、复制、删除。

### 输入框

登录输入框：

- 高度 `44px - 48px`。
- 圆角 `10px - 12px`。
- placeholder 使用弱文本色。
- 错误状态显示红色边框和一行简短错误文案。

聊天输入框：

- 支持多行输入。
- 默认高度约 `48px - 56px`。
- 最大高度约 `160px - 220px`，超过后内部滚动。
- 发送按钮放在输入区右侧或右下角。
- 上传视频、粘贴链接等附属操作应放在输入区左侧或输入框下方工具列。

### 消息

用户消息：

- 使用浅品牌背景或深色主色背景。
- 如果使用深色背景，文字必须为白色或近白色。
- 宽度不超过主内容区的 `80%`，手机端不超过 `88%`。

AI 消息：

- 以可读性优先，可以使用白色背景、轻边框或无气泡正文。
- 长内容需要良好段落间距。
- 分析结果中的标题层级要明显，但不要使用过大字号。

系统/状态消息：

- 用较小字号和弱文本色。
- 例如“正在分析视频”“已生成摘要”“分析失败，请重试”。
- 不要占据过多视觉重量。

### 视频分析结果

视频分析界面可能展示以下信息：

- 视频标题或文件名。
- 视频时长、尺寸、来源类型。
- 摘要。
- 分段脚本。
- 镜头/画面分析。
- 商品卖点。
- 受众、痛点、转化钩子。
- 建议改写文案。

展示要求：

- 长结果优先使用清晰小标题、分组、折叠区。
- 关键指标可以使用轻量标签或小型信息条。
- 不要把所有字段做成密集表格。
- 复制、导出、重新生成等操作应放在结果块右上角或底部工具区。

### 侧栏

桌面端侧栏：

- 宽度建议 `240px - 280px`。
- 背景比主内容略深或略浅，以形成区分。
- 历史会话标题单行截断。
- 当前会话有明确选中态。

手机端侧栏：

- 作为抽屉出现。
- 宽度建议 `82vw`，最大 `320px`。
- 背后使用半透明遮罩。
- 点击遮罩或返回按钮关闭。

## 8. 响应式规范

必须优先保证手机端可用。

断点建议：

```css
--bp-sm: 480px;
--bp-md: 768px;
--bp-lg: 1024px;
--bp-xl: 1280px;
```

手机端要求：

- 不出现横向滚动。
- 输入区固定底部，并适配 `env(safe-area-inset-bottom)`。
- 顶部栏高度建议 `52px - 60px`。
- 可点击区域最小高度 `44px`。
- 消息列表底部预留足够空间，避免被输入框遮挡。
- 长链接、长文件名、长英文单词必须能换行或截断。
- 弹窗在小屏应变成底部 sheet 或接近全屏。

桌面端要求：

- 主消息容器居中。
- 大屏不要把正文拉得过宽。
- 侧栏、顶部栏、输入区的边界要清晰但不厚重。

## 9. 动效与交互

动效要轻，不影响效率。

推荐：

- hover/focus 过渡：`120ms - 180ms`
- 抽屉/弹窗进入：`180ms - 240ms`
- loading 状态使用简洁 spinner、点状输入态或进度文字。

避免：

- 大幅度位移动画。
- 页面进入时复杂动画。
- 无限循环装饰动画。

交互状态必须完整：

- hover
- active
- focus-visible
- disabled
- loading
- error
- selected/current

## 10. 图标规范

- 优先使用统一图标库，例如 lucide。
- 图标线宽保持一致，通常 `1.75px - 2px`。
- 常见操作使用图标按钮：新建、菜单、上传、发送、复制、下载、删除、设置。
- 不要混用多套视觉风格不同的图标。
- 图标按钮必须有清晰的可访问名称。

## 11. 可访问性

基本要求：

- 文本与背景对比度要足够。
- 表单输入必须有 label，视觉上可以隐藏但语义上必须存在。
- 键盘可以完成登录、输入、发送、打开菜单、关闭弹窗。
- focus-visible 状态必须清楚。
- 错误提示不能只依赖颜色表达。
- loading 状态不要让用户误以为页面卡死。

## 12. 文案风格

文案要短、直接、偏工具感。

推荐：

- “登录”
- “新建对话”
- “发送”
- “上传视频”
- “粘贴链接”
- “正在分析”
- “复制结果”
- “重新分析”

避免：

- “开启你的智能视频洞察之旅”
- “让 AI 帮你释放无限创意”
- “点击这里体验强大的功能”

错误提示示例：

- “请输入账号。”
- “密码不能为空。”
- “登录失败，请检查账号或密码。”
- “视频链接无效。”
- “分析失败，请稍后重试。”

## 13. CSS Token 建议

后续实现时建议优先抽象为 token，避免颜色和间距散落在组件里。

  --bg-muted: #F1F4F8;
  --border-subtle: #E3E7EE;

  --text-primary: #111827;
  --text-secondary: #4B5563;
  --text-muted: #8A94A6;

  --brand: #2563EB;
  --brand-hover: #1D4ED8;
  --brand-active: #1E40AF;
  --brand-soft: #EAF1FF;

## 14. 实现检查清单

后续 agent 完成 UI 后，应至少检查：

- 登录页和聊天页都能在桌面端正常显示。
- 手机端宽度 `360px - 430px` 下无横向滚动。
- 聊天输入区不会遮挡最后一条消息。
- 长消息、长链接、长文件名不会撑破布局。
- 按钮、输入框、菜单、弹窗都有 hover/focus/disabled/loading/error 等必要状态。
- 页面没有大面积单一紫色、蓝紫渐变、深蓝黑、米黄色或过度装饰背景。
- 用户可以不看说明文字就理解主要操作。
- 界面第一屏就是可用的登录或聊天体验，而不是营销介绍页。
